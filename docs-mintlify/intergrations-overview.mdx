Tracer investigates alerts by querying the same tools your engineers use. Connect your stack so Tracer can pull logs, check recent deploys, map dependencies, and deliver findings where your team already works.

## Before you begin

Make sure you have:

- A Tracer account - you can create one at [app.tracer.cloud/sign-up](http://app.tracer.cloud/sign-up)
- Access credentials for the tools you want to connect
- Permissions to create API keys

## Connect integrations

We recommend connecting the following tools:

**Observability.** Tracer reads your logs, metrics, traces, and alerts to build investigation hypotheses. Connect Grafana and Prometheus.

**Infrastructure.** Tracer maps your environment and dependencies across clouds and clusters. Connect AWS.

**Communication.** Tracer delivers investigation reports where your team already works. Connect Slack.

**Knowledge.** Tracer references your runbooks, investigation guides, and architecture docs so it knows your systems the way your team does. Upload docs from your device.

## Configure integrations

Each integration follows a simple, guided flow. Let’s look at Datadog as an example. 

1. Name - Helps distinguish between different instances
2. API Key - in Datadog, create a new API key and copy and paste into Tracer. 
3. Application Key - In Datadog, create a new Application Key and copy and paste into Tracer.
